================================================================================
    SPÉCIFICITÉS TECHNIQUES
================================================================================

ENCODAGE :
----------
- Tous les fichiers XML utilisent l'encodage UTF-8
- Déclaration XML : <?xml version="1.0" encoding="utf-8" ?>

FORMAT XML :
------------
- Balise racine : <LanguageData>
- Syntaxe : <CleDef.element>Traduction</CleDef.element>
- Commentaires XML pour l'organisation : <!-- Section -->

CONVENTIONS DE TRADUCTION :
---------------------------

1. Titres et noms propres :
   - Dragons anciens : "dragon ancien" (minuscules)
   - Noms de créatures : conservés en anglais quand ce sont des noms propres
     (Nergigante, Kirin, Velkhana, etc.)

2. Interface utilisateur :
   - Tutoiement évité, forme neutre privilégiée
   - Clarté et concision pour les menus

3. Descriptions :
   - Traduction littéraire mais fidèle au sens
   - Adaptation culturelle quand nécessaire
   - Maintien du ton du jeu (aventure, chasse)

4. Variables :
   - Conservation des marqueurs de variable : {0}, {1}, etc.
   - Ordre des variables peut être changé selon la syntaxe française

COMPATIBILITÉ :
---------------

Compatible avec :
- Toutes les versions de RimWorld 1.3, 1.4, 1.5
- Mods de contenu (animaux, armes, etc.)
- Vanilla Expanded Series
- Combat Extended (non testé)

Potentiellement incompatible avec :
- Autres mods de traduction française pour Monster Hunter RimWorld
  (un seul doit être actif à la fois)

================================================================================
    NOTES POUR LES DÉVELOPPEURS
================================================================================

CONTRIBUER AU MOD :
-------------------

1. Structure à respecter :
   - Utiliser l'encodage UTF-8
   - Suivre la convention XML de RimWorld
   - Commenter les sections pour la lisibilité

2. Tester les modifications :
   - Lancer RimWorld avec le mod modifié
   - Vérifier que les traductions apparaissent correctement
   - Tester avec une partie existante ET une nouvelle partie

3. Soumettre les changements :
   - Fork le repository GitHub
   - Créer une branche pour vos modifications
   - Pull Request avec description détaillée

CONVENTIONS DE NOMMAGE :
------------------------

Fichiers DefInjected :
    - Nom identique au fichier Defs source
    - Ex: Items_Food_MH.xml traduit Items_Food_MH.xml du mod original

Fichiers Keyed :
    - Nom descriptif de la catégorie
    - Ex: Incidents.xml, UI.xml, Notifications.xml

OUTILS RECOMMANDÉS :
--------------------

- Éditeur XML : Visual Studio Code, Notepad++, Sublime Text
- Plugin XML : XML Tools (VS Code)
- Validation : RimWorld Mod Development Tools
- Test : RimWorld Developer Mode (Options → Activer le mode développeur)

================================================================================
    FOIRE AUX QUESTIONS (FAQ)
================================================================================

Q : Le mod ralentit-il le jeu ?
R : Non, les mods de traduction n'ont aucun impact sur les performances.

Q : Puis-je utiliser ce mod avec des sauvegardes existantes ?
R : Oui, absolument ! Il peut être ajouté ou retiré à tout moment.

Q : La traduction est-elle complète ?
R : La version 1.0 couvre l'essentiel (UI, incidents, objets de base).
    Les créatures et armes seront traduites dans les prochaines versions.

Q : Comment signaler une erreur de traduction ?
R : Créez une issue sur GitHub ou contactez via Discord.

Q : Puis-je modifier la traduction pour mon usage personnel ?
R : Oui ! Éditez les fichiers XML dans le dossier du mod.

Q : Le mod est-il compatible avec les DLC ?
R : Oui, il déclare la compatibilité avec Royalty, Ideology, Biotech, Anomaly.

Q : Pourquoi certains textes restent-ils en anglais ?
R : Soit ils ne sont pas encore traduits (travail en cours), soit il s'agit
    de textes provenant d'autres mods non traduits.
